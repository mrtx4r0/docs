ネットワークコマンドチートシート

linuxカーネルのルーティングテーブルを表示する
```bash
ip route
```
説明
ネットワークパケットを
・どのIFへ送るか
・どのゲートウェイへ送るか
を決める仕組み（＝ルーティングテーブル）を確認できる

出力例
```bash
default via 100.64.1.1 dev wlp2s0
100.64.1.0/24 dev wlp2s0 proto kernel scope link src 100.64.1.100
```

ip routeで出力される1行の基本的な構造
```
<宛先ネットワーク>  dev <インターフェース>  <追加情報>
```

説明
1行目
・default
デフォルトゲートウェイであることを示す
・via 100.64.1.1
パケットを「100.64.1.1に渡す」
・dev wlp2s0
Wi-Fiデバイス(NIC)から送る

2行目
・100.64.1.0/24
宛先ネットワーク100.64.1.0/24(LAN内)に対するルートは
・dev wlp2s0
Wi-Fiデバイスwlp2s0を使って届ける
・prote kernel
このルートはカーネルによって追加されたもの
・scope link
このルートはlink(同一ネットワーク)として扱われる
・src 100.1.64.100
このルートを使う場合は100.64.1.100を送信元IPとして使う

出力内容を図で表すと以下のイメージ
```
[PC]
   |
   +-- wlp2s0 ----> (LAN内へ直通) 100.64.1.0/24
   |
   +-- wlp2s0 ----> (LAN外へ) default via 100.64.1.1 → インターネット
```
