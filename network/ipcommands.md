ネットワークコマンドチートシート

linuxカーネルのルーティングテーブルを表示する
```bash
ip route
```
説明<br>
ネットワークパケットを<br>
・どのIFへ送るか<br>
・どのゲートウェイへ送るか<br>
を決める仕組み（＝ルーティングテーブル）を確認できる<br>

出力例
```bash
default via 100.64.1.1 dev wlp2s0
100.64.1.0/24 dev wlp2s0 proto kernel scope link src 100.64.1.100
```

ip routeで出力される1行の基本的な構造
```
<宛先ネットワーク>  dev <インターフェース>  <追加情報>
```

説明<br>
1行目<br>
・default<br>
デフォルトゲートウェイであることを示す<br>
・via 100.64.1.1<br>
パケットを「100.64.1.1に渡す」<br>
・dev wlp2s0<br>
Wi-Fiデバイス(NIC)から送る<br>

2行目<br>
・100.64.1.0/24<br>
宛先ネットワーク100.64.1.0/24(LAN内)に対するルートは<br>
・dev wlp2s0<br>
Wi-Fiデバイスwlp2s0を使って届ける<br>
・prote kernel<br>
このルートはカーネルによって追加されたもの<br>
・scope link<br>
このルートはlink(同一ネットワーク)として扱われる<br>
・src 100.1.64.100<br>
このルートを使う場合は100.64.1.100を送信元IPとして使う<br>

出力内容を図で表すと以下のイメージ
```
[PC]
   |
   +-- wlp2s0 ----> (LAN内へ直通) 100.64.1.0/24
   |
   +-- wlp2s0 ----> (LAN外へ) default via 100.64.1.1 → インターネット
```
